# Quality Gate Standards

Jede Phase muss folgende Quality Gates erfüllen, bevor die nächste Phase beginnt:

## Business Analyst Phase
- [ ] Klare Problembeschreibung vorhanden
- [ ] Geschäftsziele definiert
- [ ] Erfolgskriterien messbar
- [ ] Stakeholder identifiziert
- [ ] Geschäftliche Constraints dokumentiert
- [ ] Handoff-Dokument erstellt
- [ ] Benutzer-Freigabe erhalten

## Requirements Engineer Phase
- [ ] Alle funktionalen Requirements definiert
- [ ] Nicht-funktionale Requirements spezifiziert
- [ ] Requirements sind testbar
- [ ] Akzeptanzkriterien vorhanden
- [ ] Traceability Matrix erstellt
- [ ] Handoff-Dokument erstellt
- [ ] Benutzer-Freigabe erhalten

## Architect Phase
- [ ] Architektur-Design vollständig
- [ ] ADRs für wichtige Entscheidungen erstellt
- [ ] ARC42 Dokumentation aktualisiert
- [ ] Technische Risiken identifiziert
- [ ] Alle Requirements adressiert
- [ ] Architektur-Diagramme erstellt
- [ ] Handoff-Dokument erstellt
- [ ] Benutzer-Freigabe erhalten

## Code Phase
- [ ] Alle Features implementiert
- [ ] Unit Tests geschrieben (>80% Coverage)
- [ ] Integration Tests vorhanden
- [ ] Code Standards eingehalten
- [ ] Code dokumentiert
- [ ] Commits folgen Konventionen
- [ ] Handoff-Dokument erstellt
- [ ] Benutzer-Freigabe erhalten

## Debug Phase
- [ ] Alle Tests bestehen
- [ ] Keine kritischen Bugs
- [ ] Code Quality Metriken erfüllt
- [ ] Alle Requirements verifiziert
- [ ] Dokumentation vollständig
- [ ] Performance-Tests bestanden
- [ ] Finaler Bericht erstellt
